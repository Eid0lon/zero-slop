# Step 02: Prevent

## Objective

Block generic UI generation before code exists.

## Actions

1. Load `references/prevention-protocol.md`.
2. Analyze the brief or target for risky generic language.
3. Inspect existing tokens/components when a project target exists.
4. Draft the prevention contract:
   - user
   - job
   - domain expectation
   - density
   - visual voice
   - palette/type/radius/motion
   - component/state plan
   - copy rules
   - asset/data requirements
   - accessibility constraints
   - forbidden defaults
5. If economy mode is false, run the judge panel on the contract before code generation.
6. Revise the contract until it passes or block generation.

## Output

```markdown
# Prevention Contract

Status:
Preset:
Dials:
Judge gate:

## Required Decisions

## Forbidden Defaults

## Acceptance Gates
```

## Next Step

When the contract passes, generate or edit UI according to the contract, then load `step-01-detect.md` and continue through judging and verification.
