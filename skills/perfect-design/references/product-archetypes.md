# Product Archetypes

Use the closest archetype to set expectations. Mix only when the product truly spans domains.

## Operational SaaS

Expected:

- Dense but orderly information.
- Sidebar/topbar only if it supports repeated work.
- Tables, filters, inline actions, command menu, saved views, bulk actions.
- Compact typography, clear status language, stable row heights.
- Empty/error/loading states tied to the user's job.

Fails:

- Hero marketing composition inside the app.
- Oversized cards for every metric.
- Decorative motion or imagery.
- Low-density dashboards for power users.

## Developer Tool

Expected:

- Code, logs, tokens, API requests, diffs, deploys, traces, or config visible.
- Monospace used as data, not decoration.
- Copy is exact and command-oriented.
- Command palette and keyboard shortcuts are strong candidates.
- Docs/product surfaces feel connected.

Fails:

- Vague "ship faster" copy without code proof.
- Fake terminal blocks.
- Glossy hero with no artifact.

## Dashboard

Expected:

- Named decision at the top.
- Filters close to affected data.
- Metrics with units, periods, deltas, thresholds.
- Charts with labels, legends, accessible colors, and table fallback when needed.
- Tables for comparison and action.

Fails:

- Four random KPI cards.
- Unlabeled charts.
- Color-only statuses.
- Data hidden behind decorative spacing.

## Commerce

Expected:

- Product image/detail quality.
- Findability: filters, sorting, categories, counts, applied state.
- Price, shipping, returns, availability, variants, and total cost clarity.
- Honest checkout and specific validation.

Fails:

- Fake scarcity.
- Product cards without comparison attributes.
- Hidden costs.
- Generic product grids.

## Portfolio

Expected:

- Work evidence above traits.
- Case studies with role, constraints, process, outcome, artifacts.
- Typography and rhythm express the person or studio.
- Navigation makes work easy to inspect.

Fails:

- "I build beautiful digital experiences" hero.
- Skill cards replacing work.
- Faux SaaS marketing sections.

## Editorial / Brand / Awwwards-Style

Expected:

- Strong art direction, sequence, image selection, typography, and pacing.
- Interactions support narrative or exploration.
- Mobile and performance are still respected.

Fails:

- Motion as spectacle without meaning.
- Scrolljacking that hides content.
- Tiny low-contrast type.
- Usability sacrificed to trend.
