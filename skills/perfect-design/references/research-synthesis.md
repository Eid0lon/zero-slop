# Research Synthesis

Use this as compressed research memory. It is not a style moodboard; it is a set of product design constraints derived from a large web research sweep across official docs, design systems, high-end product teams, accessibility sources, and critique material.

## Sources Consulted

- Vercel Design Engineering and Web Interface Guidelines: design engineering blends aesthetic and technical skill; polish includes reusable primitives, performance, cross-browser behavior, inclusive input, accessibility, crisp borders, nested radii, and chart accessibility.
- Linear Method and Linear UI refresh: purpose-built product, small reviewable changes, designer-engineer tension, hierarchy, balance, density, and focus.
- Raycast manual/API: keyboard-first workflows, fast action surfaces, command/action separation, list navigation, forms, and predictable shortcuts.
- Awwwards evaluation system: design, usability, creativity, and content are separate scoring dimensions; expressive work still needs usability and content.
- WCAG 2.2/WAI: accessibility is a shared standard; focus indicators, text alternatives, non-text contrast, keyboard behavior, and testability are hard gates.
- Baymard ecommerce research: filtering, sorting, product presentation, checkout clarity, and form detail materially affect completion.
- Shopify Polaris, GitHub Primer, Atlassian, Carbon, Material, Apple HIG, Radix, React Aria: mature systems couple foundations, components, content, accessibility, and implementation.
- web.dev/Core Web Vitals: stable media dimensions, responsive images, CLS, and animation performance shape perceived quality.
- Resend and Supabase public design writing: strong design is coherent across marketing, support, brand, product, docs, and code.

## Premium Patterns

1. Product proof beats decoration.
   - Show the actual product, workflow, object, code, data, state, or artifact.
   - Use visual drama only when it clarifies the domain or story.

2. Density is contextual.
   - Operational tools should be compact and scannable.
   - Editorial/brand work can breathe, but must still guide the eye.

3. Hierarchy is more than font size.
   - Use spacing, alignment, contrast, grouping, position, copy specificity, and control weight.
   - Equal cards with equal headings usually flatten meaning.

4. "Premium" is coherence, not luxury styling.
   - Palette, type, radius, iconography, motion, content, and data must belong to one product.
   - Fewer stronger decisions beat many decorative effects.

5. Interaction quality is visible craft.
   - Keyboard paths, focus states, loading/empty/error/success states, undo/recovery, and disabled explanations matter.
   - A static pretty screen is not enough.

6. Accessibility is part of the design language.
   - Contrast, focus, target size, labels, semantic controls, and reduced motion are not post-processing tasks.

7. Performance is visual quality.
   - Layout shift, late-loading media, font reflow, and heavy blur/filter effects make expensive interfaces feel cheap.

8. Copy must expose the system model.
   - Labels should name the object and next action.
   - Errors should explain what happened, why it matters, and how to recover.

## Translation Rules

When inspired by:

- Vercel: use crisp systems, technical proof, restrained surfaces, performance, precise UI text.
- Linear: use hierarchy, density, focus, fast workflows, contextual actions, deliberate product model.
- Raycast: use keyboard-first ergonomics, command/action surfaces, milliseconds mindset, native-feeling restraint.
- Awwwards: use creativity, composition, typography, and interaction only when the domain can support expressive exploration.

Never copy brand identity. Translate principles into the user's product context.
