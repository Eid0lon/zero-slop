# Self-Coaching

Use this when the design feels merely "nice" or when you are stuck.

## The 12 Questions

1. What exact user is this for?
2. What is the primary job on this screen?
3. What object is the interface organized around?
4. What must be visible in the first 5 seconds?
5. What would a user compare, decide, create, approve, or recover from?
6. Which content is real proof, and which content is filler?
7. What does the current hierarchy ask the eye to do first, second, third?
8. Which visual choices are domain-specific rather than trendy?
9. What state is missing: loading, empty, error, success, disabled, selected, focus?
10. Can the primary flow work by keyboard?
11. What will break or overlap at 375px wide?
12. What would the most brutal judge call embarrassing?

## Decision Ladder

If the UI feels generic:

1. Replace abstract claims with product artifacts.
2. Replace equal cards with workflow, comparison, timeline, table, or annotated product view.
3. Tighten hierarchy before changing colors.
4. Create semantic tokens before adding effects.
5. Remove decorative motion; add stateful feedback.
6. Replace fake proof with neutral demo labels or real evidence.
7. Recheck mobile and focus.

## Premium Without Costume

Premium is not:

- black background
- big serif headline
- purple gradient
- expensive-looking whitespace
- 3D object
- glass panels

Premium is:

- fewer, sharper decisions
- exact content
- stable layout
- coherent system
- purposeful interaction
- credible proof
- accessibility that feels native
- performance that feels immediate

## Brutal Self-Review

Before judging, say:

```text
This would fail if:
- ...
```

Fill at least 5 concrete failure modes, then fix any that are true.
