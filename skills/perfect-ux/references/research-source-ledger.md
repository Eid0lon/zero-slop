# Research Source Ledger

This file records the main source families used to shape Perfect UX. Load it when a task needs research justification, citations, or a reminder of where a rule came from.

## Usability Research

- Nielsen Norman Group, Ten Usability Heuristics: visibility of system status, match to real world, user control, consistency, error prevention, recognition, flexibility, minimalist design, error recovery, help.
  - https://www.nngroup.com/articles/ten-usability-heuristics/
- Nielsen Norman Group, usability testing sample-size guidance: small formative tests find many obvious issues early; iterate rather than waiting for a perfect large study.
  - https://www.nngroup.com/articles/why-you-only-need-to-test-with-5-users/
- Nielsen Norman Group articles on mental models, task analysis, cognitive load, information architecture, error messages, progressive disclosure, and forms.
  - https://www.nngroup.com/articles/

## Accessibility Standards

- WCAG 2.2 and WAI resources: keyboard, focus appearance, target size, dragging alternatives, accessible authentication, labels, contrast, text alternatives, and robust semantics.
  - https://www.w3.org/TR/WCAG22/
  - https://www.w3.org/WAI/WCAG22/quickref/
- WAI-ARIA Authoring Practices Guide: reference patterns for dialogs, menus, tabs, comboboxes, listboxes, accordions, and disclosure widgets.
  - https://www.w3.org/WAI/ARIA/apg/
- WAI tips for designing, writing, and developing for accessibility.
  - https://www.w3.org/WAI/tips/

## Service And Content Design

- GOV.UK Service Standard: understand users and needs, solve a whole problem, make services simple, accessible, secure, and measurable.
  - https://www.gov.uk/service-manual/service-standard
- GOV.UK Design Principles: start with user needs, do the hard work to make it simple, iterate, build for inclusion.
  - https://www.gov.uk/guidance/government-design-principles
- GOV.UK Design System and Service Manual: forms, validation, error summaries, labels, hints, task lists, buttons, content, and accessibility.
  - https://design-system.service.gov.uk/
  - https://www.gov.uk/service-manual

## Platform And Design Systems

- Apple Human Interface Guidelines: clarity, deference, depth, feedback, consistency, direct manipulation, accessibility, motion, and platform expectations.
  - https://developer.apple.com/design/human-interface-guidelines/
- Material Design: components, navigation, adaptive layouts, motion, text fields, accessibility, and interaction states.
  - https://m3.material.io/
- Microsoft Fluent: cross-platform design, accessibility, input, focus, motion, and component behavior.
  - https://fluent2.microsoft.design/
- Shopify Polaris: commerce/admin UX, forms, empty states, content, accessibility, and action patterns.
  - https://polaris.shopify.com/
- IBM Carbon, Atlassian Design System, GitHub Primer, Adobe Spectrum, and Salesforce Lightning Design System: mature examples of foundations, components, content rules, accessibility, and system-level consistency.
  - https://carbondesignsystem.com/
  - https://atlassian.design/
  - https://primer.style/
  - https://spectrum.adobe.com/
  - https://www.lightningdesignsystem.com/

## Commerce UX

- Baymard Institute: ecommerce UX research around product lists, filters, sorting, forms, mobile commerce, checkout, account creation, validation, and trust.
  - https://baymard.com/

## Performance And Frontend UX

- web.dev Core Web Vitals: LCP, CLS, INP, responsiveness, layout stability, media dimensions, and perceived performance.
  - https://web.dev/articles/vitals
  - https://web.dev/articles/inp
  - https://web.dev/articles/cls
  - https://web.dev/articles/lcp

## Accessible Primitives

- React Aria, Radix UI, Ariakit, and Headless UI: accessible component behavior, keyboard support, focus management, and ARIA patterns.
  - https://react-spectrum.adobe.com/react-aria/
  - https://www.radix-ui.com/primitives
  - https://ariakit.org/
  - https://headlessui.com/

## Human-AI UX

- Google People + AI Guidebook: user-centered AI, mental models, feedback, control, explainability, data, and failure planning.
  - https://pair.withgoogle.com/guidebook/
- Microsoft Human-AI Interaction Guidelines: timing, explanations, user control, feedback, correction, and graceful failure for AI systems.
  - https://www.microsoft.com/en-us/research/project/guidelines-for-human-ai-interaction/
- NIST AI Risk Management Framework: validity, reliability, safety, security, transparency, explainability, privacy, fairness, and accountability context.
  - https://www.nist.gov/itl/ai-risk-management-framework

## UX Laws And Cognitive Principles

- Laws of UX summary of Fitts's Law, Hick's Law, Jakob's Law, Miller's Law, Peak-End Rule, Tesler's Law, Von Restorff Effect, and related heuristics.
  - https://lawsofux.com/

Use these sources as constraints, not as decoration. Cite them when explaining why a UX gate exists.
