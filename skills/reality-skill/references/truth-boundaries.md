# Truth Boundaries

Use this file when a real service, backend, dataset, credential, or production integration is unavailable.

## Principle

The UI may be local, mocked, or sample-backed. It may not pretend to be production-backed.

Truth is a product feature. Users should know what will persist, what will affect real systems, and what is illustrative.

## Allowed Labels

Use clear labels where trust matters:

- `Sample data`
- `Local draft`
- `Saved in this browser`
- `Sandbox mode`
- `Demo provider`
- `Requires API credentials`
- `Export unavailable`
- `Mock response`
- `Offline mode`

Do not bury these labels in tiny legal copy if the action looks production-grade.

## Authentication

Allowed:

- local demo session explicitly labeled
- disabled OAuth buttons with setup requirements
- real auth through existing configured provider

Blocked:

- sign-in accepts anything while implying security
- fake OAuth success
- fake roles/permissions
- fake audit logs

## Payments

Allowed:

- disabled payment with provider requirement
- cart/price calculator labeled estimate/sample
- sandbox checkout only if actual sandbox integration exists

Blocked:

- fake charge success
- fake receipts/order IDs as real
- fake tax/shipping certainty

## AI

Allowed:

- real model call through existing configured API
- deterministic local mock labeled as sample/mock
- missing-key state with setup instructions

Blocked:

- canned generated output pretending to be live
- fake progress followed by unrelated result
- claims of privacy/security not backed by code

## Analytics

Allowed:

- sample analytics labeled sample
- locally computed metrics from current data
- real analytics when provider code and data source exist

Blocked:

- random KPIs
- fake growth percentages
- decorative charts without source, units, or time range

## Integrations

Allowed:

- disconnected state
- disabled connect button with required configuration
- sandbox provider with clear label
- real connect flow using existing provider code

Blocked:

- fake synced timestamp
- fake imported data
- fake OAuth popup/success

## Exports

Allowed:

- actual CSV/JSON/PDF/image/text export of current data
- copy-to-clipboard with feedback
- disabled export with reason

Blocked:

- Export button that only toasts success
- generated file with stale/fake data

## Notifications

Allowed:

- local in-app notifications generated by actual actions
- disabled email/SMS/push with provider requirement

Blocked:

- fake sent email/SMS/push claims
- fake invite delivery

## Compliance And Security

Allowed:

- modest UI copy describing current local behavior
- explicit "not configured" or "sample" status

Blocked:

- SOC 2, HIPAA, GDPR, encryption, audit trail, access control, or retention claims not backed by project code and configuration
